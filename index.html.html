Unit 1 — Introduction to C++ & Functions

17 questions covering OOP concepts, C++ basics, variables, functions, and memory

Procedure-oriented programming (POP) means writing programs as a list of steps (procedures/functions) executed one by one.

  • Focus: HOW to do a task — step by step
  • Data: Shared globally — any function can access it (less safe)
  • Examples: C, Pascal, COBOL, FORTRAN
  • Problem: Hard to manage in large programs — data gets messy
💡 Think of it like a cooking recipe — step 1, step 2, step 3... straight from top to bottom.

OOP organizes programs around objects — like real-world things that have both data and behavior.

  • Object = bundles data + functions together
  • Focus: WHAT something is, not just how to do steps
  • Examples: C++, Java, Python

4 main concepts of OOP:

  • Encapsulation — Wrapping data and functions together, hiding internal details
  • Inheritance — Child class reuses parent class code
  • Polymorphism — One name, many forms
  • Abstraction — Show only what's necessary, hide complexity
✅ OOP is more secure, reusable, and easier to maintain than POP.

Class = Blueprint or template (like the design of a car)

Object = An actual thing created from the blueprint (like the actual car)

class Car {
public:
    string color;
    int speed;
    void drive() { cout << "Car is driving!"; }
};

int main() {
    Car myCar;          // myCar is an object of class Car
    myCar.color = "Red";
    myCar.drive();
}
  • Class defines: attributes (color, speed) and behaviors (drive, stop)
  • Object uses those attributes and behaviors
  • Many objects can be created from ONE class

Reusability means writing code once and using it multiple times — no need to rewrite the same logic.

  • Achieved through Inheritance — child class gets parent class features automatically
  • Saves time — don't rewrite existing code
  • Fewer bugs — already-tested code is reused
  • Easy maintenance — fix in one place, fixed everywhere
💡 Example: A "Vehicle" class with speed and fuel — Car, Truck, and Bike all inherit from it. You write the code once for Vehicle and all three get it!
  • Object-Oriented — Supports classes and objects
  • Encapsulation — Hides data inside classes
  • Inheritance — Child reuses parent's code
  • Polymorphism — Same name, different behavior
  • Abstraction — Hides complexity, shows only necessary info
  • Exception Handling — Handle errors without crashing
  • Templates — Write one function/class for all data types
  • Fast — Compiled language, faster than Java
  • Both POP + OOP — Supports both procedural and OOP styles

new and delete are used for dynamic memory allocation — allocating memory while the program is running.

int* p = new int;   // Allocate memory for one int
*p = 50;
cout << *p;
delete p;            // Free the memory

// For arrays:

  • delete — frees that memory when done
  • Without delete → memory leak (memory fills up wastefully)
  • Very important for large programs

A function is a named block of code that does a specific task. You call it whenever you need it.

#include <iostream>
using namespace std;

// Function definition
int add(int a, int b) {
    return a + b;
}

void greet(string name) {
    cout << "Hello, " << name << "!" << endl;
}

int main() {
    int result = add(5, 3);    // Function call
    cout << "Sum = " << result << endl;  // Output: Sum = 8

    greet("Riya");             // Output: Hello, Riya!
    return 0;
}
  • Functions make code reusable — call the same function many times
  • Functions make code organized — each function does one job

Pass by Value

A COPY of the value is sent. Original is NOT changed.

Pass by Reference

The ACTUAL variable is sent. Original IS changed.

// Pass by VALUE — original safe
void byValue(int x) {
    x = 100;  // Only copy changes
}

// Pass by REFERENCE — original changes
void byRef(int &x) {
    x = 100;  // Actual variable changes
}

int main() {
    int a = 5;
    byValue(a);  cout << a;  // Still 5
    byRef(a);    cout << a;  // Now 100
}

The return statement sends a value back from a function to where it was called.

int square(int n) {
    return n * n;  // Sends the result back
}

int main() {
    int result = square(4);  // result = 16
    return 0;  // 0 means "program ended successfully"
}
  • Functions with int, float, etc. MUST return a value
  • Functions with void do NOT return a value
  • return 0; in main() = program ran successfully
  • Execution stops as soon as return is encountered

Default arguments are pre-set values for parameters — used when the caller doesn't pass a value.

void greet(string name = "World", int times = 1) {
    for(int i = 0; i < times; i++)
        cout << "Hello, " << name << "!" << endl;
}

int main() {
    greet();              // Hello, World!
    greet("Riya");        // Hello, Riya!
    greet("Aman", 3);     // Hello, Aman! (3 times)
}
  • Default arguments must be at the right end of the parameter list
  • Makes functions more flexible

An inline function tells the compiler: "paste this function's code directly where it's called" instead of making a separate function call.

inline int square(int x) {
    return x * x;
}

int main() {
    cout << square(5);  // Compiler replaces with: 5 * 5
}
  • Purpose: Avoids the overhead (time cost) of calling a function
  • Best for small, simple functions (1-3 lines)
  • Makes program faster
  • Not suitable for large functions — code size increases

Unit 2 — Classes, Constructors & Objects

15 questions covering class structure, constructors, access specifiers, arrays, and destructors

A class is like a blueprint. It has data members (variables) and member functions (methods).

class Student {
private:               // Hidden from outside
    int rollNo;
    string name;

public:                // Accessible from outside
    void setData(int r, string n) {
        rollNo = r;
        name = n;
    }
    void display() {
        cout << rollNo << " - " << name << endl;
    }
};

int main() {
    Student s;           // Creating object
    s.setData(1, "Riya");
    s.display();         // Output: 1 - Riya
}

Member functions are functions that are defined inside a class. They work on the class's data.

  • They keep data and behavior together (encapsulation)
  • They control how data is accessed and modified
  • Can be private (internal use only) or public (accessible by anyone)
  • Make code more organized and secure
✅ Without member functions, data inside a class cannot be manipulated properly.

Access specifiers control who can access the data and functions in a class.

💡 Default in C++ class is private. Default in struct is public.

A constructor is a special function that is automatically called when an object is created. It initializes the object's data.

class Car {
public:
    string color;
    int speed;

    Car() {            // Constructor — called automatically
        color = "White";
        speed = 0;
        cout << "Car is created!" << endl;
    }
};

int main() {
    Car c;   // Automatically calls Car() → "Car is created!"
}
  • Same name as the class
  • No return type (not even void)
  • Called automatically at object creation
class Student {
    string name; int age;
public:
    // Default Constructor — no parameters
    Student() { name = "Unknown"; age = 0; }

    // Parameterized Constructor — takes values
    Student(string n, int a) { name = n; age = a; }

    // Copy Constructor — copies from another object
    Student(Student &s) { name = s.name; age = s.age; }

    // Default Argument Constructor — has defaults
    Student(string n = "No Name", int a = 18) { name = n; age = a; }
};

int main() {
    Student s1;                   // Default
    Student s2("Riya", 20);       // Parameterized
    Student s3(s2);               // Copy
}

Having multiple constructors in the same class with different parameters. C++ chooses the right one based on arguments passed.

class Box {
public:
    int length, width, height;

    Box() { length = width = height = 1; }              // No args
    Box(int l) { length = l; width = height = 1; }      // 1 arg
    Box(int l, int w, int h) { length=l; width=w; height=h; } // 3 args
};

int main() {
    Box b1;          // Calls first constructor
    Box b2(5);       // Calls second
    Box b3(3,4,5);   // Calls third
}

Converting one type to another — either basic type to class or class to basic type.

class Celsius {
    float temp;
public:
    Celsius(float t) { temp = t; }

    // Convert Celsius to Fahrenheit (class to basic)
    operator float() {
        return (temp * 9/5) + 32;
    }
};

int main() {
    Celsius c(100);
    float f = c;    // Conversion: 100°C → 212°F
    cout << f;
}

Conversion can be done using conversion constructors or conversion functions.

Private data cannot be accessed directly from outside the class. We use getter and setter functions.

Student* s2 = new Student; // Heap — need delete

Unit 3 — Operator Overloading & Inheritance

9 questions covering operator overloading, inheritance types, and containership

What is it? Giving new meaning to existing operators (+, -, *, etc.) when used with objects of a class.

class Complex {
    float real, imag;
public:
    Complex(float r, float i) { real=r; imag=i; }

    // Overloading the + operator
    Complex operator+(Complex c) {
        return Complex(real + c.real, imag + c.imag);
    }

    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(2, 3), c2(1, 4);
    Complex c3 = c1 + c2;   // Uses overloaded + operator
    c3.display();            // Output: 3 + 7i
}
  • Importance: Makes code more natural and readable. e.g., c1 + c2 is clearer than c1.add(c2)
  • Types of operators that can be overloaded:
    • Unary — ++, --, -(negative), !
    • Binary — +, -, *, /, %, =
    • Relational — ==, !=, <, >
    • Stream — <<, >> (for cin/cout with objects)
  • Role: Allows objects to behave like built-in types in expressions
💡 You CANNOT overload: :: (scope), . (member access), .* (member pointer), ?: (ternary)

Inheritance means a child class automatically gets the properties and methods of a parent class.

class Animal {           // Parent (Base) class
public:
    string name;
    void eat() { cout << name << " is eating." << endl; }
    void sleep() { cout << name << " is sleeping." << endl; }
};

class Dog : public Animal {   // Child (Derived) class
public:
    void bark() { cout << name << " says: Woof!" << endl; }
};

int main() {
    Dog d;
    d.name = "Tommy";
    d.eat();    // Inherited from Animal
    d.sleep();  // Inherited from Animal
    d.bark();   // Dog's own method
}
✅ Reuse code without rewriting it. Dog automatically gets eat() and sleep().

protected members are accessible in the class itself AND its child classes, but NOT from outside.

class Person {
protected:
    string name;    // Accessible in child classes
    int age;
public:
    void setInfo(string n, int a) { name=n; age=a; }
};

class Student : public Person {
public:
    void display() {
        cout << name << ", Age: " << age; // ✅ Works!
    }
};

int main() {
    Student s;
    s.setInfo("Riya", 20);
    s.display();
    // s.name = "Aman"; ❌ ERROR — protected, not public!
}
💡 protected = "family secret" — parent shares with child, but not with strangers.

The child class redefines a function that already exists in the parent class with the same name and parameters.

class Animal {
public:
    void sound() { cout << "Some generic sound" << endl; }
};

class Dog : public Animal {
public:
    void sound() {        // Overrides Animal's sound()
        cout << "Woof! Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() {        // Overrides Animal's sound()
        cout << "Meow!" << endl;
    }
};

int main() {
    Dog d; d.sound();    // Woof! Woof!
    Cat c; c.sound();    // Meow!
}

Hybrid inheritance is a combination of two or more types of inheritance (e.g., hierarchical + multiple).

        A          ← Base class
       / \
      B   C        ← Hierarchical
       \ /
        D          ← Multiple (inherits B and C)
  • Importance: Maximum code reuse, most flexible structure
  • Problem: Can cause Diamond Problem — D inherits A's members twice (via B and via C)
  • Solution: Use virtual base class
class B : virtual public A { };  // Virtual inheritance
class C : virtual public A { };
class D : public B, public C { }; // Diamond solved

Containership (also called composition or "Has-A" relationship) means one class contains an object of another class as its member.

class Engine {
public:

};

class Car {
public:

    Car(string b, int hp) : e(hp) { brand = b; }

    void drive() {
        e.start();  // Using contained object
        cout << brand << " is driving." << endl;
    }
};

Unit 4 — Polymorphism, Templates & Exception Handling

9 questions covering polymorphism, virtual functions, templates, and exception handling

Polymorphism = "Poly" (many) + "morph" (forms) = One name, many behaviors.

TypeWhen decidedHow
Compile-time (Static)At compile timeFunction overloading, Operator overloading
Runtime (Dynamic)While runningVirtual functions
// Runtime Polymorphism example:
class Shape {
public:
    virtual void draw() { cout << "Drawing shape" << endl; }
};

class Circle : public Shape {
public:
    void draw() { cout << "Drawing Circle ⭕" << endl; }
};

class Square : public Shape {
public:
    void draw() { cout << "Drawing Square ■" << endl; }
};

int main() {
    Shape* s;
    Circle c;  Square sq;

    s = &c;  s->draw();    // Drawing Circle
    s = &sq; s->draw();    // Drawing Square
}

A pointer that stores the memory address of an object instead of a basic variable.

class Student {
public:
    string name;
    void display() { cout << "Name: " << name << endl; }
};

int main() {
    Student s;
    s.name = "Riya";

    Student* ptr = &s;    // Pointer to object s

    ptr->name = "Aman";   // Access with arrow operator (->)
    ptr->display();       // Name: Aman

    // Dynamic object:
    Student* ptr2 = new Student;
    ptr2->name = "Zara";
    ptr2->display();
    delete ptr2;           // Free memory!
}
  • Use . for normal objects, -> for pointer to object
  • Essential for polymorphism — base class pointer can point to derived class objects

A virtual function is declared in the base class and overridden in derived classes. It enables runtime polymorphism — the correct version is called based on the actual object type.

class Animal {
public:
    virtual void sound() {     // virtual keyword!
        cout << "Some sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() {             // Override
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() {             // Override
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* a;
    Dog d;  Cat c;

    a = &d;  a->sound();   // Woof! (correct!)
    a = &c;  a->sound();   // Meow! (correct!)
}
✅ Without virtual, Animal's sound() would be called for both. With virtual, the right version is called.

Static Binding

Function decided at compile time. Normal functions. Faster but less flexible.

Dynamic Binding

Function decided at runtime. Virtual functions. Flexible — correct version called based on actual object.

How they connect: When a function is declared virtual, C++ uses a vtable (virtual table) — a lookup table that stores the correct function address for each class. At runtime, the program checks this table to call the right function.

Animal* a = new Dog();   // Base pointer, derived object
a->sound();   
// Static binding: would call Animal::sound()
// Dynamic binding (virtual): calls Dog::sound() ✅
💡 Dynamic binding is what makes runtime polymorphism actually work!

this is a special pointer that points to the current object — the object that called the function.

class Student {
    string name;
    int age;
public:
    // Problem: parameter name same as member name
    void setData(string name, int age) {
        this->name = name;   // this->name = member, name = parameter
        this->age  = age;
    }

    // Returning the current object itself
    Student& increment() {
        age++;
        return *this;        // Return current object
    }

    void display() {
        cout << name << ", " << age << endl;
    }
};

int main() {
    Student s;
    s.setData("Riya", 20);
    s.increment().increment();   // Method chaining
    s.display();    // Riya, 22
}
  • Resolve ambiguity when member names = parameter names
  • Return current object for method chaining
  • Pass current object to another function

Templates let you write one function or class that works for any data type.

Type 1: Function Template

template <typename T>
T findMax(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    cout << findMax(3, 7);       // Works for int → 7
    cout << findMax(3.5, 2.1);   // Works for float → 3.5
    cout << findMax('a', 'z');   // Works for char → z
}

Type 2: Class Template

template <typename T>
class Stack {
    T data[100];
    int top;
public:
    Stack() { top = -1; }
    void push(T x) { data[++top] = x; }
    T pop() { return data[top--]; }
};

int main() {
    Stack<int> intStack;      // Stack of integers
    Stack<string> strStack;   // Stack of strings
    intStack.push(10);
    strStack.push("Hello");
}
✅ Without templates, you'd write separate add() for int, float, double... Templates solve this!

Exception handling lets you handle runtime errors gracefully — the program doesn't crash, instead it handles the error properly.

3 keywords: try, catch, throw

int divide(int a, int b) {
    if(b == 0)
        throw "Division by zero!";    // throw the error
    return a / b;
}

int main() {
    try {                              // try risky code
        int result = divide(10, 0);
        cout << result;
    }
    catch(const char* msg) {           // catch the error
        cout << "Error: " << msg << endl;
    }

    cout << "Program continues..." << endl;
}
// Output:
// Error: Division by zero!
// Program continues...
  • try — wraps the risky code
  • throw — signals that an error occurred
  • catch — handles the error

Multiple catch blocks allow handling different types of errors in different ways.

void testErrors(int x) {
    if(x == 1) throw 100;           // Throw int error
    if(x == 2) throw "File error";  // Throw string error
    if(x == 3) throw 3.14;          // Throw float error
}

int main() {
    for(int i = 1; i <= 4; i++) {
        try {
            testErrors(i);
        }
        catch(int e) {
            cout << "Integer error: " << e << endl;
        }
        catch(const char* e) {
            cout << "String error: " << e << endl;
        }
        catch(float e) {
            cout << "Float error: " << e << endl;
        }
        catch(...) {
            cout << "Unknown error!" << endl;  // Catch-all
        }
    }
}
  • Catches are checked in order — first matching catch is executed
  • catch(...) = catch ALL types — always put it last

Polymorphism is one of the most important concepts in modern programming. Here's why it matters:

  • Flexibility — One interface works for many types. Write once, apply everywhere.
  • Extensibility — Add new classes without changing existing code
  • Maintainability — Cleaner, less code duplication
  • Real-world modeling — Things behave differently based on context
  • Framework foundation — Most UI frameworks, game engines rely on polymorphism
// Real-world example: Payment system
class Payment {
public:
    virtual void pay(float amount) = 0;  // Pure virtual
};

class CreditCard : public Payment {
public:
    void pay(float a) { cout << "Paid Rs." << a << " via Credit Card"; }
};

class UPI : public Payment {
public:
    void pay(float a) { cout << "Paid Rs." << a << " via UPI"; }
};

class Cash : public Payment {
public:
    void pay(float a) { cout << "Paid Rs." << a << " in Cash"; }
};

// All use the same pay() — but behavior differs!
void processPayment(Payment* p, float amount) {
    p->pay(amount);  // Polymorphism in action
}
✅ Without polymorphism, you'd need if-else or switch statements for every payment type — ugly and hard to extend!
